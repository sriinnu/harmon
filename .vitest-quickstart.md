# Vitest Quick Start Guide

## Running Tests

```bash
# Watch mode (development)
pnpm test

# Run once (CI/CD)
pnpm test:run

# With coverage
pnpm test:coverage

# With UI
pnpm test:ui

# In specific package
cd packages/harmon-core && pnpm test

# Filter by file pattern
pnpm test -- engine

# Filter by test name
pnpm test -- -t "should validate"
```

## Writing Tests

### Basic Test Structure

```typescript
import { describe, it, expect } from 'vitest'
import { myFunction } from './myFunction.js'

describe('myFunction', () => {
  it('should return expected value', () => {
    const result = myFunction('input')
    expect(result).toBe('expected')
  })
})
```

### Testing Zod Schemas (harmon-protocol)

```typescript
import { describe, it, expect } from 'vitest'
import { MySchema } from './schemas.js'

describe('MySchema', () => {
  it('should accept valid data', () => {
    const valid = { field: 'value' }
    expect(() => MySchema.parse(valid)).not.toThrow()
  })

  it('should reject invalid data', () => {
    const invalid = { field: 123 }
    expect(() => MySchema.parse(invalid)).toThrow()
  })
})
```

### Async Tests

```typescript
it('should handle async operations', async () => {
  const result = await asyncFunction()
  expect(result).toBeDefined()
})
```

### Mocking

```typescript
import { vi } from 'vitest'

const mockFn = vi.fn()
mockFn.mockReturnValue('mocked value')

expect(mockFn()).toBe('mocked value')
expect(mockFn).toHaveBeenCalled()
```

### Setup/Teardown

```typescript
import { beforeEach, afterEach } from 'vitest'

beforeEach(() => {
  // Setup before each test
})

afterEach(() => {
  // Cleanup after each test
})
```

## Coverage

Coverage reports are generated in `coverage/` directory:
- `coverage/index.html` - Browse detailed HTML report
- `coverage/coverage-final.json` - JSON for tooling
- Console output - Text summary

## Common Matchers

```typescript
expect(value).toBe(expected)           // Strict equality (===)
expect(value).toEqual(expected)        // Deep equality
expect(value).toBeTruthy()             // Truthy check
expect(value).toBeFalsy()              // Falsy check
expect(value).toBeNull()               // Null check
expect(value).toBeUndefined()          // Undefined check
expect(value).toBeDefined()            // Defined check
expect(array).toContain(item)          // Array contains
expect(array).toHaveLength(n)          // Array length
expect(string).toMatch(/pattern/)      // Regex match
expect(fn).toThrow()                   // Function throws
expect(fn).toHaveBeenCalled()          // Mock called
expect(fn).toHaveBeenCalledWith(args)  // Mock called with
```

## Debugging

```bash
# Run with verbose output
pnpm test -- --reporter=verbose

# Run single test file
pnpm test -- path/to/test.test.ts

# Show full error details
pnpm test -- --reporter=verbose --no-coverage
```

## Tips

1. Use descriptive test names that explain what is tested
2. Keep tests focused - one assertion per test when possible
3. Use `describe` blocks to group related tests
4. Mock external dependencies to isolate units
5. Test both success and failure cases
6. Use `beforeEach`/`afterEach` for common setup/cleanup
7. Avoid test interdependencies - each should run independently
8. Write tests alongside code for better maintainability

## Package-Specific Notes

### harmon-core
- Path alias: `@sriinnu/harmon-protocol` available
- Focus on engine, ranking, adaptation logic

### harmon-store
- Timeout: 10s for DB operations
- Consider using in-memory SQLite for tests
- Test migrations separately

### harmon-crypto
- Test encryption/decryption round-trips
- Verify key generation
- Test error cases

### harmon-protocol
- Test Zod schema validation
- Test type inference
- Verify parsing edge cases

### harmond
- Timeout: 15s for integration tests
- All workspace packages aliased
- Test daemon commands/events
- Consider spinning up test daemon
