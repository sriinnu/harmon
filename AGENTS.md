# Agent Guidelines

These guidelines apply to all new and modified code in this repo.

## Structure

- Keep packages modular and single-purpose. Prefer small, focused modules over large files.
- Keep files at or under 300 lines. Split by concern when a file grows beyond this.
- Keep functions small and easy to scan. Split complex logic into helpers.

## Documentation

- Add JSDoc for exported functions, classes, and types.
- Use brief inline comments only where the logic is non-obvious.
- Keep comments accurate and up to date.
